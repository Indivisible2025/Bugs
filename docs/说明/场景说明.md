# 场景说明

## 什么是场景

场景是 Bugs 中的一个独立上下文单元。每个场景有自己独立的记忆、配置和行为。

## 场景类型

| 类型 | 示例 | 检测方式 |
|:----|:----|:--------|
| 项目场景 | `/home/you/project-a/` | 自动：进入目录时检测 .bugs/ 或 BUG.md |
| Git 仓库场景 | 任何 git 仓库 | 自动：检测 .git/ 目录（可配置开关） |
| 学习场景 | 「量子计算」对话 | 手动创建 /scene 命令 |
| 日常场景 | general | 默认场景 |
| 临时场景 | 一次对话 | 用完即弃 |

## 场景自动检测

场景切换需要手动执行 `/scene <name>` 命令，或通过 API 端点 `POST /api/scenes/switch` 自动切换。

> 注意：当前版本不支持 pwd 变化自动检测。该功能已在 SceneManager 中预留接口，计划后续版本实现。

## 场景切换代价

| 过程 | 耗时 |
|:----|:---:|
| 配置重载 | < 1ms |
| 旧记忆压缩归档 | < 20ms |
| 新记忆加载 | < 80ms |
| **总计** | **< 100ms** |

## 嵌套场景

```
/home/you/project-a/sub-project/
    ↓
sub-project 有自己的 BUG.md → 场景 "sub-project"
但它也是 project-a 的子目录
    ↓
规则：最近匹配优先，子目录场景独立
多目录可映射到同一场景（通过配置声明 scene_paths）
```

## 防抖

快速连续 cd 操作（如脚本遍历目录）→ 200ms 内只执行最后一次切换，避免频繁 IO。
