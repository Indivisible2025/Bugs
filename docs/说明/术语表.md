# 术语表

## 核心角色

| 术语 | English | 含义 |
|:----|:-------|:----|
| Bugs | — | 项目名，双关：程序 Bug + 星空虫族 |
| **Overmind** | Overmind | 主Agent，星空虫族的最高意志，负责思考规划与子Agent派遣 |
| **SubAgent** | SubAgent | 子Agent（兵虫），由 Overmind 派遣的轻量执行单元 |
| **调度引擎** | Scheduler | 程序组件，负责任务排队、分发、负载均衡、故障转移 |
| **记忆网格** | Memory Mesh | 分级记忆存储系统（全局/场景概要/完整场景） |
| **信任引擎** | Trust Engine | 记忆信用评分系统，速率驱动 + 交叉验证 |
| **冥想系统** | Meditate | 后台记忆整理系统，经验提炼+关系发现+清理 |
| **模块注册表** | Module Registry | 管理所有 BugModule 的注册和发现 |
| **MCP 服务器** | McpServer | Model Context Protocol 服务器，JSON-RPC 2.0 |
| **LSP 客户端** | LspClient | 语言服务器协议客户端，代码诊断+定义+引用 |
| **Cron 调度器** | CronScheduler | 后台定时任务调度器 |
| **配置监视器** | ConfigWatcher | 配置文件变更自动检测 |
| **技能注册表** | SkillRegistry | SKILL.md 技能系统，OpenClaw 兼容格式 |

## 记忆系统

| 术语 | English | 含义 |
|:----|:-------|:----|
| 记忆网格 | Memory Mesh | 分级记忆存储系统 |
| 全局记忆 | Global Memory | 永远存在的顶层记忆 |
| 场景概要 | Scene Summary | 永远存在的场景压缩摘要 |
| 完整记忆 | Full Memory | 仅当前场景加载的完整记忆 |
| 语义压缩 | Semantic Compression | 信任值驱动的记忆压缩策略 |
| 信任引擎 | Trust Engine | 记忆信用评分系统 |
| 信任值 | Trust Value | 记忆的可信度，-∞ 到 +∞ |
| 交叉验证 | Cross-validation | 多个独立来源确认同一事实 |
| 去重 | Deduplication | 相似记忆合并，保留来源标签链 |
| 时间衰减 | Time Decay | 长期未被验证的记忆信任缓慢下降 |

## 调度系统

| 术语 | English | 含义 |
|:----|:-------|:----|
| 分片 | Shard | 调度引擎的并行处理单元，每 CPU 核心一个 |
| 老化机制 | Aging | 队列中等待时间越长优先级越高的策略 |
| 多维评分 | Multi-dimensional Scoring | 综合局部性/能力/负载/亲和分配 Executor |
| 局部性 | Locality | 刚完成同类任务的 Executor 优先分配 |
| 能力匹配 | Capability Matching | 已加载所需资源的 Executor 优先分配 |
| 任务种类 | TaskKind | 推理/工具调用/文件操作/浏览器/知识检索 |
| 死信 | Dead Letter | 重试用尽无法恢复的任务 |
| 指数退避 | Exponential Backoff | 每次重试延迟翻倍的重试策略 |
| 优雅关闭 | Graceful Shutdown | 排空所有运行中任务后再退出的关闭方式 |
| 桶 0 逃逸 | Bucket 0 Escape | 超时任务强制进入紧急优先级的机制 |

## 场景系统

| 术语 | English | 含义 |
|:----|:-------|:----|
| 场景 | Scene | 独立的上下文（如项目A、课题B、日常对话） |
| 场景切换 | Scene Switch | 自动检测 pwd 并切换对应记忆 |
| 防抖 | Debounce | 快速连续切换时的延迟保护（200ms） |

## 权限系统

| 术语 | English | 含义 |
|:----|:-------|:----|
| 审批级别 | Approval Level | 6 级审批：全部审批/信任审批/计划/自动/白名单/全部放行 |
| 保护路径 | Protected Path | 任何审批级别下不自动放行的系统关键路径 |
| 信任审批 | Trust-based Approval | 信任值超过阈值自动放行文件操作 |

## 生态与扩展

| 术语 | English | 含义 |
|:----|:-------|:----|
| 插件 | Plugin | 第三方运行时扩展，添加新能力 |
| 技能 | Skill | 指令扩展（Markdown），教 Agent 用法 |
| MCP 协议 | Model Context Protocol | JSON-RPC 2.0 标准，对接外部服务 |
| 工具扩展 | Tool Extension | 轻量级第三方工具注册（MCP/HTTP/CLI） |
| 竞品兼容 | Competitor Compatibility | 原生支持 MCP/SKILL.md/CLAUDE.md 格式 |

## 配置与接口

| 术语 | English | 含义 |
|:----|:-------|:----|
| BUG.md | — | 核心 Agent 规则文件（Markdown + YAML 头） |
| ROLE.md | — | 工作模式定义文件 |
| MEMORY.md | — | 记忆和信任机制配置文件 |
| config.json | — | 结构化 JSON5 配置（支持注释和尾逗号） |
| 继承规则 | Override Chain | 全局 → 场景 → 本地，逐层覆盖 |
| 热重载 | Hot Reload | 修改配置文件后无需重启即可生效 |
| 本地端口 | Local Port | `127.0.0.1:8742`，仅本机访问 |
| 远程域名 | Remote Domain | `0.0.0.0:8742`，Token 认证，多设备访问 |
