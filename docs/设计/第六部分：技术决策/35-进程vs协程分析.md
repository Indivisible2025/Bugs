# 进程 vs 协程：取舍分析

## 本质区别

| 维度 | 进程隔离路径 | 协程 (Bugs) |
|:----|:-----------------|:----------|
| 内存 | 独立页表 ~10MB/个 | 共享地址 ~200 bytes/个 |
| 创建 | ~2ms (fork+exec) | ~100ns (push to tokio) |
| 切换 | ~1μs (内核态) | ~100ns (用户态) |
| 崩溃隔离 | ✅ | ⚠️ |
| 安全隔离 | ✅ | ❌ |
| 并发上限 | ~数万 | **无上限** |

## Bugs 的代价和补救

**代价① 崩溃隔离不足：**

```rust
// 每个子Agent 用 catch_unwind 包装
let result = std::panic::catch_unwind(|| subagent_task());
// panic 不影响进程，标记任务失败即可
```

**代价② 内存无隔离：**

```rust
// TaskBudget 限制单任务
tokio::select! {
    _ = subagent_task => {},
    _ = tokio::time::sleep(cpu_limit) => { handle.abort(); }
}
```

**代价③ 进程挂=全挂：** 看门狗 + state.json + 降级模式 + 自动恢复。完整韧性体系见 19-安全设计.md。

## 结论

Bugs 选协程是因为定位是**单用户个人 Agent**。牺牲隔离性换取 500KB 内存 + 100ns 调度。如果未来需要多租户，用 Docker 容器补。
