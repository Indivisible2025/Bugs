# 进程模型与部署

## 进程模型

```
bugs（CLI 入口）
    │
    ├── bugs start → 启动守护进程（后台 daemon）
    │       └── 启动后返回 0，守护进程继续运行
    │
    ├── bugs（TUI）→ 如果守护进程未运行，自动启动
    │       ├── 守护进程 → bugs-core（后台 task）
    │       └── TUI → ratatui（主线程）
    │       两者在同一进程内，通过 tokio channel 通信
    │
    └── bugs --web → 启动 HTTP 服务器（同一进程）
            ├── HTTP → axum
            └── core → bugs-core
```

## 单进程 vs 多进程

| 模式 | 场景 | 进程模型 |
|:----|:----|:--------|
| TUI 模式 | 终端本地使用 | 守护进程 + TUI 同进程，tokio task 分离 |
| Web 模式 | 浏览器／远程 | 守护进程 + HTTP 同进程 |
| GUI 模式 | 桌面窗口 | 守护进程 + egui 同进程 |
| Android | 手机 | Kotlin 独立进程，通过 HTTP/WS 连核心 |
| 远程连接 | 手机连 PC | 手机 APK → HTTP/WS → PC 上 bugs daemon |

## 编译打包

```
# 多平台构建
cargo build --release --target x86_64-unknown-linux-gnu
cargo build --release --target x86_64-pc-windows-msvc
cargo build --release --target aarch64-unknown-linux-gnu

# 单二进制产物
target/release/bugs → 内含 TUI + daemon + HTTP 全部功能

# Docker（可选）
FROM scratch
COPY bugs /bugs
ENTRYPOINT ["/bugs", "--web"]
```

## systemd 服务

```ini
[Unit]
Description=Bugs AI Agent Daemon
After=network.target

[Service]
ExecStart=/usr/local/bin/bugs daemon
Restart=always
RestartSec=10
WatchdogSec=30

[Install]
WantedBy=multi-user.target
```
