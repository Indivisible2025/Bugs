# 版本控制

## 概述

Bugs 有两套并行存在的版本号：**内部版本**供开发团队追踪代码演进，**外部版本**供用户识别发布版本。

---

## 内部版本

### 格式

```
<内部大版本号>-<发行类型>-<开发内部版本>-<开发版本号>

示例：
    1-stable-0-1      首次稳定版
    2-beta-1-5        大版本2的第1个beta，第5次构建
    3-dev-0-42         大版本3开发中，第42次构建
```

| 段 | 含义 | 范围 | 说明 |
|:---|:----|:----|:----|
| 内部大版本号 | 架构/API 重大变更 | 1, 2, 3... | 递增，不降 |
| 发行类型 | stable / beta / dev | 三选一 | 标识当前分支的稳定性 |
| 开发内部版本 | 同一大版本内的迭代 | 0, 1, 2... | 从 0 开始，stable 发布时归零 |
| 开发版本号 | 每次构建递增 | 1, 2, 3... | CI 自动递增，不可手动修改 |

### 分支策略

```
mainline
    │
    ├── 1-stable ──── 1-stable-0-1, 1-stable-0-2, ...
    ├── 2-dev    ──── 2-dev-0-1, 2-dev-0-2, ...
    │   ├── 2-beta    ──── 2-beta-1-1, 2-beta-1-2, ...
    │   └── 2-stable  ──── 2-stable-2-1, ...
    └── 3-dev    ──── 3-dev-0-1, ...
```

### 合并规则

```
分支 A（内部大版本号 = X）→ 分支 B（内部大版本号 = Y）

X == Y  → ✅ 可以直接合并
X != Y  → ❌ 不能直接合并
         → 需要先升级/降级内部大版本号至相同
        
升级场景：
    分支 A（v2-stable）→ 合并到主分支（v3-dev）
        → 先将 v2 的内容升级至兼容 v3（更新 API 调用、数据格式等）
        → 内部版本号从 2-x-x-x 升级到 3-0-0-1
        → 合并

降级场景（cherry-pick 修复）：
    分支 A（v3-dev）的某修复 → 合并到 v2-stable
        → 将修复改写为兼容 v2 的代码
        → 内部版本号用 v2 的版本号 + 递增的开发版本号
        → 合并
```

## 外部版本

### 格式

```
<年>.<月>.<日>-<发行类型>-<内部版本号>

示例：
    2026.05.26-stable-1      2026年5月26日发布的稳定版，内部版本 1
    2026.06.01-beta-2        2026年6月1日发布的beta版，内部版本 2
    2026.06.15-dev-3         2026年6月15日发布的开发版，内部版本 3
```

| 段 | 含义 | 说明 |
|:---|:----|:----|
| 年 | 发布年份 | 四位数字 |
| 月 | 发布月份 | 两位数字，补零 |
| 日 | 发布日期 | 两位数字，补零 |
| 发行类型 | stable / beta / dev | 与内部版本的发行类型对应 |
| 内部版本号 | 对应的内部大版本号 | 省略了发行类型和开发版本号，只保留大版本 |

### 使用方式

```
$ bugs --version
bugs 2026.05.26-stable (internal: 1-stable-0-42)

WebUI 关于页面：
    Bugs v2026.05.26-stable
```

## 版本嵌入

```rust
// build.rs 编译时生成
pub const VERSION_INTERNAL: &str = env!("BUGS_INTERNAL_VERSION");  // 1-stable-0-42
pub const VERSION_EXTERNAL: &str = env!("BUGS_EXTERNAL_VERSION");  // 2026.05.26
pub const BUILD_DATE: &str = env!("BUGS_BUILD_DATE");
pub const GIT_HASH: &str = env!("BUGS_GIT_HASH");
```

## 编译特性差异

发行类型决定编译时开启的功能集：

| 功能 | dev | beta | stable |
|:----|:---:|:----:|:------:|
| **日志级别** | trace（全量） | debug | info |
| **性能剖析** | ✅ | ✅ | ❌ |
| **内存追踪** | ✅ | ✅ | ❌ |
| **死锁检测** | ✅ | ✅ | ❌ |
| **匿名使用统计** | ❌ | ✅ | ❌ |
| **崩溃报告** | ✅ | ✅ | ✅ |
| **调试断言** | ✅ | ❌ | ❌ |
| **热重载检测增强** | ✅ | ✅ | ❌ |

```rust
// build.rs：根据环境变量 BUGS_RELEASE_TYPE 设置 cfg
// 不传默认为 dev
let release_type = env::var("BUGS_RELEASE_TYPE").unwrap_or("dev".into());
println!("cargo:rustc-cfg=release_{}", release_type);

// 源码条件编译
#[cfg(release = "dev")]
{ /* 全量日志 + 性能剖析 */ }

#[cfg(release = "stable")]
{ /* 仅 info + 崩溃报告 */ }
```
