# 版本控制

## 格式

```
v<年>.<月>.<日>-<大版本>-<小版本>--<发行类型>-<开发代号>-<开发迭代版本>

v2026.05.27-1-1--dev-overmind-1
```

| 段 | 含义 | 递增规则 |
|:---|:----|:----|
| 年.月.日 | 发布日期 | 当天日期，不变 |
| 大版本 | 功能性/更改性更新 | 重大功能、架构变更、API 不兼容时递增 |
| 小版本 | 修复性更新 | 同一天内的 Bug 修复、小优化递增 |
| 发行类型 | stable / beta / dev | 随发布阶段变化 |
| 开发代号 | 每个大版本的代号 | 大版本递增时换新代号 |
| 开发迭代版本 | 开发过程中的构建次数 | 每次构建/发布递增 |

> `--` 是分隔符，左边是「用户关心的版本信息」，右边是「开发追踪信息」。

---

## 段详解

### 大版本（功能性/更改性更新）

对应重大里程碑：

```
1 → 首个大版本（核心引擎 + 基础功能）
2 → 重大功能扩展（如：多 Agent 协作 v2、新存储引擎）
3 → 架构重构（如：进程模型改为 actor 模型）
...
```

大版本递增时，小版本归零，开发迭代重新计数。

### 小版本（修复性更新）

同一天内修了几个 Bug：

```
v2026.05.27-1-1--dev-overmind-1   ← 首次构建
v2026.05.27-1-2--dev-overmind-2   ← 修了一个崩溃
v2026.05.27-1-3--dev-overmind-3   ← 又修了一个内存泄漏
```

日期变化时小版本通常归 1，重大修复例外可保留计数。

### 发行类型

| 类型 | 含义 | 分支 |
|:----|:----|:----|
| `dev` | 开发中，功能不完整，可能崩溃 | `dev` |
| `beta` | 功能冻结，收集反馈，基本稳定 | `beta` |
| `stable` | 生产就绪，经过充分测试 | `main` |

### 开发代号

每个大版本一个代号，用英文单词，体现该版本的主题：

```
大版本1 → overmind    （主控引擎）
大版本2 → swarm      （多 Agent 协作）
大版本3 → weave      （深度集成）
```

### 开发迭代版本

从 1 开始，每次构建递增。大版本变更时归 1。

## 完整示例

```
v2026.05.27-1-1--dev-overmind-1      首次开发构建
v2026.05.27-1-2--dev-overmind-2      修复了一个 Bug
v2026.05.28-1-1--dev-overmind-3      第二天继续开发
v2026.05.30-1-1--beta-overmind-4     Beta 发布
v2026.05.30-1-2--beta-overmind-5     Beta 修复
v2026.06.01-1-1--stable-overmind-6   首个稳定版 🎉
v2026.06.01-1-2--stable-overmind-7   稳定版热修复

v2026.06.15-2-1--dev-swarm-1         大版本 2 开始
v2026.07.01-2-1--beta-swarm-12       大版本 2 Beta
v2026.08.01-2-1--stable-swarm-20      大版本 2 稳定版
```

---

## 分支策略

```
main    ──── stable   生产就绪
beta    ──── beta     功能冻结，收集反馈
dev     ──── dev      日常开发，最新代码
```

合并方向：

```
dev  →  beta  →  main
  ↑               │
  └─── hotfix ────┘  （稳定版紧急修复反向合并）
```

## 版本在代码中的使用

```bash
$ bugs --version
bugs v2026.05.27-1-1--dev-overmind-1
```

```rust
// build.rs 编译时嵌入
pub const VERSION: &str = env!("BUGS_VERSION");
// v2026.05.27-1-1--dev-overmind-1

// 解析版本信息
pub fn release_type() -> &str { /* dev */ }
pub fn major_version() -> u32 { /* 1 */ }
```

Cargo.toml 中直接使用完整版本号：

```toml
[workspace.package]
version = "2026.05.27-1-1--dev-overmind-1"
```
