# 定时任务与心跳

## 一、Cron 调度器

内建定时任务引擎：

| 定时任务 | 默认频率 | 说明 |
|:--------|:------:|:----|
| 记忆压缩 | 每天 03:00 | 压缩所有场景完整记忆 |
| 衰减扫描 | 每天 04:00 | 全量记忆衰减计算 |
| 主动验证 | 每天 | 选取最不稳定记忆询问用户 |
| 自动备份 | 每周日 05:00 | 导出到 backup 目录 |
| 健康检查 | 每 30s | 检查所有子Agent 状态 |
| 模块检查 | 每天 | 检查模块更新 |

## 二、心跳机制（借鉴 OpenClaw）

双层心跳：

```
组件层：子Agent → 调度引擎 每 5s → 含状态和任务数
        15s 无心跳 → 标记 dead → 任务回收

Agent层：子Agent 长任务时每 10s 报告进度
        连续 3 次 no_progress → 主Agent 干预
```

## 三、冥想

冥想是 Agent 的后台思考时间——借鉴了业内成熟的自动记忆整理模式，重新用 Rust 实现了记忆驱动的版本。

```
触发：用户空闲 > 5min / 每天凌晨 3:00

过程：
    ① 记忆回顾：强化高频记忆衰减速率
    ② 知识组织：发现关联，建立引用
    ③ 经验提炼：多条经验类记忆合并为通用知识
    ④ 清理：低信任记忆标记"待确认"
    ⑤ 预判：根据用户行为预加载信息到缓存
```

**冥想与普通记忆整理的区别：** 冥想是信任驱动的——冥想结果直接改写增强/衰减速率，而非被动备份。冥想后可询问用户确认发现。
