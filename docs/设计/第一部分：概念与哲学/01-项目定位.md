# 项目定位

## Bugs 是什么

**Bugs 是一个 AI Agent 运行时框架**。名字取自《星船伞兵》中 Arachnids 的俗称——"Bugs"。

它的核心设计哲学一句话：

> **主Agent（Overmind）分裂任务 → 调度引擎（虫后）管理子Agent（兵虫）池 → 子Agent 并行执行。**

就像星空虫族在战场上：一个Overmind不会亲自撕咬敌人，它的价值在于指挥一群兵虫高效协作。Bugs 也一样——主 Agent 负责思考规划，子 Agent 负责并行执行。

它不像 OpenCode / Cursor 那样绑定在单个项目里，而是像 OpenClaw 一样运行在你的机器上，跨项目、跨场景、跨设备。

## 对标关系

| 类似 | 但不同 |

| OpenClaw（全局个人助手） | + 场景记忆隔离 |
| 进程隔离方案（编码助手） | + 全平台、全前端 |
| Hermes-Go（AI 框架） | + 极端低资源占用 + 极端并发 |

## 命名来源

- **Bug** = 程序漏洞（程序员的本行）
- **Bugs** = 虫族（科幻小说的蜂巢文明）
- 双关：像虫族一样高效协作，又扎根在程序员的日常里

## 一句话

> 像虫族一样组织 AI Agent，像天体计算机一样积累知识。
