# 设计哲学

## 核心原则

| # | 原则 | 含义 |
|:-:|:----|:----|
| 1 | **用程序代替 AI** | 能用算法和程序实现的功能绝不创建独立的 AI Agent |
| 2 | **主Agent + 子Agent 协作** | 主Agent（Overmind）分裂任务，子Agent（兵虫）并行执行。不是多层级多Agent，而是主人派遣模式 |
| 3 | **极端性能底线** | 低资源占用 + 子Agent 高并发是 P0，做不到就不发布 |
| 4 | **场景记忆隔离** | 不同场景的记忆严格隔离，不能串 |
| 5 | **配置驱动行为** | Agent 的行为由配置文件控制，不改代码 |
| 6 | **全局 + 本地覆盖** | 全局配置 + 场景覆盖 + 本地覆盖，三层优先级 |
| 7 | **核心无感知** | 核心不 import 任何模块代码，所有扩展通过能力注册表 |
| 8 | **一条路径** | 重构不留 shim、不保留旧路径、不堆 fallback |
| 9 | **兼容是选择** | 不默认兼容旧配置，需要显式标记迁移路径 |

## 不是特征

- ❌ 不是风味饮料
- ❌ 不是绑定项目的工具
- ❌ 不是多 Agent 大乱炖
- ❌ 不是重框架
