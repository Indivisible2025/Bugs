# 冥想系统设计

---

## 一、是什么

冥想是 Bugs 的后台记忆整理系统。不是主 Agent 在做，不是某个后台线程在做，而是**脑虫（专门的冥想子Agent）**在做。

类比星空虫族：两次征战之间，兵虫休整，脑虫整理战果，蜂巢更新集体记忆。

## 二、触发条件

| 条件 | 说明 |
|:----|:----|
| **用户空闲** | 无交互 > 5 分钟 |
| **定时** | 每天凌晨 3:00（可配置） |
| **手动** | `/meditate` 命令 |
| **场景切换后** | 退出当前场景时自动冥想整理 |

防打扰：用户正在对话时不触发。冥想是低优先级后台任务。

## 三、冥想做什么

```
① 记忆衰退扫描
   遍历所有场景记忆 → 应用衰减速率 → 更新 strength_cached
   低 trust + 久未验证 → 标记"待确认"

② 知识去重合并
   查找语义相似度 > 0.85 的记忆 → 合并为通用知识
   来源独立去重 → 合并后的增强速率按加权合并

③ 经验提炼
   同一场景多条经验类记忆 → 提取共性 → 合并为更抽象的规则
   例: "项目A的API超时5s" + "项目B的API超时3s" → "网络不太稳定"

④ 关系发现
   高频共现的记忆 → 建立引用关系
   例: "模块A依赖B" 和 "模块B依赖C" → "模块A间接依赖C"

⑤ 清理
   连续 7 天 strength < -5 → 自动删除
   连续 30 天未验证 → 降级为"休眠记忆"（不注入上下文但保留）

⑥ 预判
   分析用户近期行为 → 预加载可能需要的记忆到缓存
   例: 每天早起先查某个项目 → 预加载该场景完整记忆

⑦ 报告
   冥想完成后生成摘要：
   - 合并了 X 条记忆
   - 强化了 Y 条记忆
   - 清理了 Z 条低信任记忆
   - 发现了 N 个新关联
   - 询问用户确认："以下记忆还准确吗？"
```

## 四、算法详情

### 4.1 语义去重

```
遍历当前场景 + 全局记忆
    ↓ 按 category 分组
    ↓ 组内两两计算余弦相似度
    ↓ sim > 0.85 → 合并候选
    ↓ 合并规则：
        enhancement_rate = weighted_avg(rate_A, rate_B)
        source_count = |union(sources_A, sources_B)|
        content = 保留更完整的一方
```

### 4.2 经验提炼规则

```
同一 category=experience 且 scene_id相同 的记忆
    ↓ 如果数量 >= 3
    ↓ 提取语义共性关键词
    ↓ 生成 category=knowledge 的抽象记忆
    ↓ 新记忆 enhancement_rate = 源记忆的加权平均 × 0.5
    ↓ 源记忆仍保留（经验仍然有效）
```

### 4.3 清理规则

```
遍历所有记忆：
    strength < -5 且 last_validated > 7天 → 删除
    strength < 0 且 last_validated > 30天 → 标记为 dormant
    validation_count == 0 且 last_updated > 14天 → 标记为待确认
```

## 五、API 设计

| 端点 | 说明 |
|:----|:----|
| `POST /api/meditate` | 手动触发冥想 |
| `GET /api/meditate/status` | 查询最近冥想结果 |
| `GET /api/meditate/pending` | 查询待确认的记忆列表 |
| `POST /api/meditate/confirm` | 确认/否定某条记忆 |

### 响应示例

```json
{
  "last_run": "2026-05-27T03:00:00Z",
  "duration_ms": 2340,
  "results": {
    "merged": 3,
    "strengthened": 47,
    "cleaned": 2,
    "discovered_relations": 1,
    "pending_confirmations": 5
  }
}
```

## 六、资源约束

| 约束 | 值 |
|:----|:---|
| 最大执行时间 | 30s（超时中断） |
| CPU 占用 | < 10%（后台低优先级） |
| 内存增量 | < 50MB（只加载当前场景记忆） |
| 合并候选上限 | 每次最多处理 100 对候选 |

## 七、与现有系统的交互

```
冥想子Agent → 读取 MemoryMesh → 信任引擎计算 → 写入 MemoryMesh
                                                   ↓
                                              持久化 Store
                                                   ↓
                                              生成报告 → 通知用户（下次对话时）

冥想期间不影响：
    ✅ 主Agent 对话（冥想是独立的子Agent）
    ✅ 其他子Agent 执行
    ✅ 记忆检索
```

## 八、假性能自检

| 调度引擎犯的错 | 冥想有没有？ |
|:-------------|:----------|
| 固定权重 | ❌ 无评分 |
| 两阶段截断 | ⚠️ 合并候选100对上限是安全保护 |
| 立即重试 | ❌ 不重试，下次冥想周期继续处理 |
| 无上限 | ❌ 最大执行时间30s硬限制 |
