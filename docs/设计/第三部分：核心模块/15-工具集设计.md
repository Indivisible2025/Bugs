# 工具集设计

> 基于竞品深度分析（22-竞品深度分析.md）的结论。

---

## 一、Bugs 自带哪些工具

Bugs 有两类自带工具：**必选**（启动就有，不可卸载）和 **可选**（内置但按需启用/禁用）。

没有自带的就是模块工具（第三方提供）或 MCP 工具（通过协议对接）。

### 必选工具

| 工具 | 功能 | 为什么必选 |
|:----|:----|:----------|
| read | 读取文件内容 | 基础能力，P0 |
| write | 写入文件 | 基础能力，P0 |
| edit | 修改文件部分内容 | 基础能力，P0 |
| ls | 列出目录 | 基础能力，P0 |
| exec | 执行 Shell 命令 | 基础能力，P0 |
| process | 进程管理 | 基础能力，P0 |
| memory_search | 检索记忆网格 | Bugs 核心差异（记忆） |
| memory_store | 写入记忆网格 | Bugs 核心差异（记忆） |
| knowledge_query | 知识库查询 | Bugs 核心差异（知识宝库） |

### 可选工具

| 工具 | 功能 | 为什么可选 | 启用前提 |
|:----|:----|:----------|:--------|
| browser | 网页浏览、内容提取 | 依赖 Chromium CEF 内核，较重 | 内核已安装 |
| code_exec | 安全执行用户代码 | 依赖容器/沙箱环境 | 沙箱已配置 |

### 不自带的工具

| 工具 | 获取方式 | 为什么不自带 |
|:----|:--------|:------------|
| HTTP 请求 | MCP 协议 | 已有标准协议，不需要重复发明 |
| 文件解析（PDF/Word） | 模块 | 场景特殊，按需安装 |
| 图片处理 | 模块 | 场景特殊，按需安装 |
| 其他三方服务 | MCP / 模块 | 各取所需 |

## 二、功能分组（配置用）

分组的目的是方便在配置中批量授权/禁用：

| 分组 | 包含工具 | 自带策略 |
|:----|:--------|:--------|
| group:fs | read, write, edit, ls | 必选 |
| group:runtime | exec, process | 必选 |
| group:memory | memory_search, memory_store | 必选 |
| group:knowledge | knowledge_query | 必选 |
| group:browser | browser | 可选 |
| group:sandbox | code_exec | 可选 |

配置示例：

```json
{
  "tools": {
    "allow": ["group:fs", "group:runtime", "group:memory", "group:knowledge"],
    "deny": ["group:browser", "group:sandbox"]
  }
}
```

## 三、工具与权限的关联

> 详细权限控制见 **05-Agent权限控制设计.md**。子Agent 默认继承主Agent 的工具权限，可额外收紧。

| 自带策略 | 默认权限 | 可调节 | 子Agent 默认继承 |
|:--------|:--------|:------|:---------------|
| 必选工具 | 全允许 | 可在配置中禁用个别工具 | ✅ 全继承 |
| 可选工具 | 默认禁用，启用后全允许 | 可在配置中按组限制 | ⚠️ 继承但可收紧 |
| 模块工具 | 安装后默认允许 | 可按模块 ID 黑白名单 | ⚠️ 继承但可收紧 |
| MCP 工具 | 配置后默认允许 | 可按源限制 | ⚠️ 继承但可收紧 |
