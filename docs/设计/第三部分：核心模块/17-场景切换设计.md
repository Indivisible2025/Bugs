# 场景切换设计

---

## 一、是什么

场景切换是 Bugs 中**自动检测并切换当前工作上下文**的程序功能。它不是 AI 决策，而是基于环境信号（当前目录、打开的会话、用户指令）的规则引擎。

---

## 二、场景检测优先级

```
检测顺序（高→低）：

① 用户显式命令：/scene project-a
    ↓ 不存在
② 当前工作目录：pwd → 匹配已注册场景的路径
    ↓ 不匹配
③ 会话绑定：当前对话窗口曾绑定到某场景
    ↓ 未绑定
④ 默认场景：全局 default 场景
```

### 场景目录识别

```
当前目录是 /home/user/code/project-a/
    ↓ 向上遍历
检查 /home/user/code/project-a/ 下是否有：
    - .bugs/ 目录
    - BUG.md 文件
    - .git/ 目录（可选开关：git 仓库自动视为场景）
    ↓ 匹配到
场景 = "project-a"
    ↓ 检查是否有对应场景配置
~/.bugs/scenes/project-a/ 存在 → 加载
不存在 → 自动创建（空场景，使用全局默认配置）
```

## 三、切换流程

```
检测到场景变化（如 cd 到新目录）
    ↓
① 卸载旧场景完整记忆
    ├── 压缩归档 → 生成/更新场景概要
    ├── 持久化到磁盘
    └── 释放内存
    ↓
② 加载新场景配置
    ├── 全局 BUG.md（始终保留）
    ├── 场景 BUG.md（覆盖全局配置）
    └── 场景 config（工具权限、信任参数等）
    ↓
③ 加载新场景完整记忆
    ├── 从磁盘读取
    ├── 解压到内存
    └── 就绪
    ↓
切换完成（< 100ms 目标）
```

### 切换性能预算

| 步骤 | 耗时预算 |
|:----|:-------:|
| 配置文件重载 | < 1ms |
| 全局记忆重索引 | < 10ms |
| 完整记忆加载 | < 80ms（依赖磁盘 IO） |
| Agent 上下文刷新 | < 10ms |
| **总计** | **< 100ms** |

## 四、特殊场景

### 无关联目录

```
用户在终端聊天（不是在项目中工作）
    ↓ 场景检测无匹配
    ↓ 使用 "general" 默认场景
```

### 嵌套项目

```
用户 cd 到 /home/user/code/parent-project/sub-project/
    ↓ sub-project 有自己的 BUG.md → 场景 sub-project
    ↓ 但它也是 parent-project 的子目录
    ↓ 规则：最近匹配优先
```

### 快速切换（防抖）

```
用户连续 cd 多个目录（如脚本操作）
    ↓ 防抖：200ms 内只执行最后一次切换
    ↓ 避免频繁的磁盘 IO
```

### 跨目录共同场景

```
多个目录映射到同一场景：
    /home/user/code/project-a/frontend
    /home/user/code/project-a/backend
    ↓ 都归属于 "project-a" 场景
    ↓ 配置中声明 scene_paths: ["/home/user/code/project-a/**"]
```

---

## 五、配置结构

```json
{
  "scenes": {
    "auto_detect": true,
    "debounce_ms": 200,
    "git_as_scene": true,
    "default_scene": "general",
    "registered": {
      "project-a": {
        "paths": ["~/code/project-a/**"],
        "config_dir": "~/.bugs/scenes/project-a/"
      },
      "project-b": {
        "paths": ["~/code/project-b/**"],
        "config_dir": "~/.bugs/scenes/project-b/"
      }
    }
  }
}
```

## 六、假性能自检

| 调度引擎犯的错 | 场景切换有没有？ | 说明 |
|:-------------|:-------------|:----|
| 固定权重 | ❌ | 无评分 |
| 两阶段截断 | ❌ | 目录匹配是精确匹配 |
| 分段跳跃 | ❌ | 防抖是固定 200ms |
| 立即重试 | ❌ | 切换失败不会自动重试 |
| 无上限 | ❌ | 性能预算明确 100ms |
| 随意阈值 | ⚠️ | 200ms 防抖是经验值 |
