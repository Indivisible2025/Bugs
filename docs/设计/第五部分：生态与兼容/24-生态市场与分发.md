# 生态市场与分发

---

## 一、分发渠道

| 渠道 | 类型 | 说明 | 优先级 |
|:----|:----|:----|:-----:|
| **Git 仓库** | 去中心 | 直接 `bugs plugin install git:github.com/...` | P0 |
| **本地路径** | 开发 | `bugs plugin install ./my-plugin` | P0 |
| **MCP 注册表** | 兼容 | 发现并连接 MCP 服务器 | P1 |
| **ClawHub** | 兼容 | 直接安装 ClawHub 上发布的 OpenClaw 插件 | P2 |
| **官方仓库** | 集中 | Bugs 官方插件市场（审核制） | P3 |

P0 必须、P1 重要、P2 锦上添花、P3 远期。

---

## 二、插件发现

```
bugs plugin search keyword          → 搜索本地+远程仓库
bugs plugin search clawhub:keyword  → 搜索 ClawHub
bugs plugin search mcp:keyword      → 搜索 MCP 注册表

结果统一展示：名字 + 描述 + 来源 + 版本
```

---

## 三、版本管理

```
插件版本：语义化版本（SemVer） 1.0.0
插件声明的 Bugs 兼容版本：bugs_version = ">=0.1.0"
安装策略：
    bugs plugin install my-plugin          → 最新版
    bugs plugin install my-plugin@1.2.0    → 指定版本
    bugs plugin install my-plugin@latest   → 最新版（显式）
```

---

## 四、安全策略

```
安装前：显示插件声明需要的权限
    ├── filesystem: ["read", "write"]
    ├── network: ["outbound"]
    └── tools: ["execute_command"]
    
用户确认：安装时一次性授权
事后可收回：任何一个权限被滥用 → 用户可禁用该权限或卸载插件
```

---

## 五、更新策略

```
自动检查更新（默认关闭）
手动更新：bugs plugin update my-plugin
批量更新：bugs plugin update --all
版本锁定：bugplug.toml 中可声明 "pinned": true
```
