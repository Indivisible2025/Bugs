# 模块扩展机制

---

## 一、模块是什么

模块是 Bugs 的**第三方运行时扩展**。模块可以添加新工具、新技能(Skills)、新 Provider。与"工具扩展接口"的区别：模块是完整的独立包，工具扩展是轻量注册。

---

## 二、模块类型

| 类型 | 说明 | 示例 |
|:----|:----|:----|
| **工具模块** | 添加新的可调用工具 | 浏览器模块（第一个内置模块） |
| **技能模块** | 添加 Skill 指令包 | /review-pr 工作流 |
| **Provider 模块** | 添加新的 LLM 提供商 | Ollama、vLLM 适配器 |
| **多媒体模块** | TTS/STT/图像处理 | 语音转文字、文字转语音 |

> 浏览器是第一个模块。后续计划包括 TTS/STT 等日常和开发必备模块。

---

## 三、模块标准

### 文件结构

```
my-module/
├── bugmod.toml          ← 模块清单（必选）
├── src/                 ← 模块代码
│   └── lib.rs
├── skills/              ← 技能（可选）
│   └── SKILL.md
├── README.md
└── LICENSE
```

### bugmod.toml 格式

```toml
[module]
name = "my-module"
version = "1.0.0"
description = "一个示例模块"
author = "开发者名"
bugs_version = ">=0.1.0"

[tools]
count = 1

[permissions]
filesystem = ["read", "write"]
network = ["outbound"]
```

---

## 四、模块生命周期

```
发现 → 安装 → 启用 → 运行 → 更新 → 卸载
```

---

## 五、竞品兼容

Bugs 模块系统兼容以下格式：

| 格式 | 兼容方式 |
|:----|:--------|
| MCP 工具 | 内置 MCP 客户端 |
| OpenClaw Skills | 原生读取 SKILL.md |
| CLAUDE.md | BUG.md 兼容读取 |
