# 工具扩展接口

---

## 一、是什么

工具扩展接口是**轻量级第三方工具注册标准**，比完整插件更简单。不需要打包、安装、权限声明，只要实现接口即可注册为 Bugs 的可调用工具。

与插件的关系：

| 维度 | 工具扩展 | 完整插件 |
|:----|:--------|:--------|
| 安装方式 | 配置文件声明一行 | 需要安装、启用、生命周期管理 |
| 代码形式 | MCP Server / HTTP 端点 / 可执行文件 | 编译为 .wasm 或动态库 |
| 安全隔离 | 外部进程（无直接内存访问） | 进程内（需要沙箱） |
| 适用场景 | 集成现有服务、脚本工具 | 深度集成、性能敏感场景 |

---

## 二、三种扩展方式

### 方式 A：MCP 协议（推荐）

```
配置声明：
{
  "mcpServers": {
    "my-tool": {
      "type": "stdio",
      "command": "my-tool-server",
      "args": []
    }
  }
}
```

Executor 通过内置 MCP 客户端与远程工具通信。这是最推荐的扩展方式。

**优点：** 语言无关、标准协议、可直接复用 MCP 生态的工具。

### 方式 B：HTTP 端点

```
配置声明：
{
  "externalTools": {
    "weather": {
      "url": "http://localhost:8080/call",
      "schema": {
        "name": "weather",
        "description": "获取天气",
        "inputSchema": {
          "type": "object",
          "properties": {
            "location": { "type": "string" }
          },
          "required": ["location"]
        }
      }
    }
  }
}
```

Executor HTTP 请求调用。

**优点：** 最简单，任何 HTTP 服务都行。

### 方式 C：CLI 工具

```
配置声明：
{
  "externalTools": {
    "my-script": {
      "type": "cli",
      "command": "python3",
      "args": ["/path/to/tool.py"],
      "schema": { ... }
    }
  }
}
```

Executor 通过 STDIO 通信。

**优点：** 无需常驻服务，用完即退。

---

## 三、接口规范

无论哪种方式，工具扩展的调用模型统一为：

```
Executor → 序列化请求（JSON） → 调用扩展工具 → 等待响应（JSON） → 返回结果
```

请求格式：

```json
{
  "tool": "工具名",
  "args": { "参数1": "值1", "参数2": "值2" },
  "request_id": "唯一ID"
}
```

响应格式：

```json
{
  "request_id": "唯一ID",
  "result": { "type": "text", "text": "返回内容" },
  "error": null
}
```

## 四、安全策略

```
默认：所有外部工具都是不受信任的
启用条件：用户手动配置并且确认
执行限制：超时（默认 30s）、资源限制（内存/CPU）、网络限制（可选）
```

## 五、假性能自检

| 错 | 工具有没有？ | 说明 |
|:---|:---------|:----|
| 固定权重 | ❌ | 无评分 |
| 两阶段截断 | ❌ | 无截断 |
| 热路径优化 | ❌ | 外部工具调用不是热路径，不参与调度核心 |
