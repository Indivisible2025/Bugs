## 一、竞品生态总览

| 项目 | 扩展形式 | 协议/格式 | 分发渠道 | Bugs 兼容策略 |
|:----|:--------|:---------|:--------|:-------------|
| OpenClaw | Plugin + Skills | openclaw.plugin.json + SKILL.md | ClawHub + npm + git | 兼容 |
| MCP 标准 | MCP Server | JSON-RPC 2.0 | 无集中渠道 | 直接兼容 |

## 二、三种扩展类型辨析

### 插件 (Plugin)

| 属性 | 说明 |
|:----|:----|
| **本质** | 运行时扩展，添加新能力 |
| **形式** | 代码模块（编译或脚本） |
| **能力边界** | 可注册新工具、新渠道、新模型、新事件 |
| **安全** | 需要沙箱隔离 |
| **示例** | 浏览器插件（添加 browser 工具和页面渲染能力） |

### 技能 (Skill)

| 属性 | 说明 |
|:----|:----|
| **本质** | 指令扩展，教 Agent 怎么用现有能力 |
| **形式** | SKILL.md（YAML frontmatter + Markdown 指令） |
| **能力边界** | 不能添加新工具，只能指导已有工具的用法 |
| **安全** | 纯文本，无执行风险 |
| **加载方式** | 注入 Agent 的 system prompt |
| **示例** | hello-world 技能（教 Agent 用 echo 工具打招呼） |

### MCP 协议

| 属性 | 说明 |
|:----|:----|
| **本质** | 标准协议，对接外部服务 |
| **形式** | JSON-RPC 2.0 端点 |
| **传输** | STDIO（本地） / Streamable HTTP（远程） |
| **服务端能力** | Tools（可执行函数）/ Resources（数据源）/ Prompts（模板） |
| **客户端能力** | Sampling（请求 LLM 补全）/ Elicitation（请求用户输入） |
| **生命周期** | initialize → 能力协商 → ready |

## 三、兼容策略

### 直接兼容（原生支持，零转换成本）

| 标准 | 方式 |
|:----|:----|
| **MCP 协议** | 内置 MCP 客户端，Executor 可直接调用 MCP Server 的工具 |
| **SKILL.md** | 原生读取 Skill，注入到 BUG.md 上下文 |
| **CLAUDE.md** | BUG.md 兼容读取 CLAUDE.md 内容 |

### 格式转换（需要适配层）

| 标准 | 转换方式 |
|:----|:--------|
| **OpenCode Commands** | 命令行式 Markdown → 转为 SKILL.md 格式 |
| **LangChain Toolkits** | 通过适配器层桥接为 Bugs 工具 |

### 参考设计（不兼容，但借鉴思路）

| 标准 | 可借鉴的部分 |
|:----|:------------|
| **OpenClaw Plugin** | 生命周期管理、安全隔离、热重载 |
| **MCP** | 能力协商机制、JSON-RPC 协议 |
| **OpenCode** | 多模型 Provider 支持 |
