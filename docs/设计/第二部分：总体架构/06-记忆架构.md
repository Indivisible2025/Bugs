# 记忆架构

## 一、三层结构

```
全局记忆 (Global Memory)
    ├── 永远存在
    ├── 身份 / 偏好 / 通用知识 / 全局信任
    └── 类比星空虫族：种族本能——所有兵虫天生就懂的战斗直觉

场景概要 (Scene Summary)
    ├── 永远存在（每个场景都有）
    ├── 关键信息聚合 / 压缩摘要 / 索引
    └── 类比星空虫族：兵虫抵达新星球时快速感知到的战场环境

完整场景记忆 (Full Scene Memory)
    ├── 仅进入当前场景时加载
    ├── 完整对话链 / 信任评分 / 向量索引
    ├── 退出场景 → 压缩归档 → 更新概要
    └── 类比星空虫族：Overmind 对该星球的完整记忆
```

## 二、生命周期

| 层 | 存在范围 | 加载时机 | 退出场景后 | 大小上限 |
|:--|:--------|:--------|:----------|:--------|
| 全局记忆 | 所有场景 | 始终常驻 | 保留 | 1 MB |
| 场景概要 | 所有场景 | 始终常驻 | 保留 | 100 KB/场景 |
| 完整记忆 | 仅当前场景 | 进入时加载 | 卸载到持久层 | 50 MB/场景 |

### 压缩归档

```
完整记忆（内存中，~50MB）
    ↓ 语义压缩（提取关键决策、重要上下文、高频引用）
场景概要（~100KB，事实性摘要 + 索引）
    ↓ 持久化
磁盘存储
```

压缩策略由信任引擎驱动（详见 16-信任引擎设计.md）。

## 三、知识分类

记忆不是扁平的。每条记忆带有**类型标签**，不同类型的记忆有不同的衰减速度和检索优先级。

| 分类 | 衰减速度 | 折半期 | 示例 | 特点 |
|:----|:-------:|:-----:|:----|:----|
| **知识类** (knowledge) | 0.999/天 | ~2 年 | "Rust 的 Vec 是堆分配的" | 客观事实，几乎不变 |
| **习惯类** (habit) | 0.99/天 | ~2 月 | "用户偏好 Tab 缩进，缩进宽度为 4" | 用户偏好，可能改变 |
| **经验类** (experience) | 0.95/天 | ~2 周 | "上次调用此 API 超时了，建议加 5s 超时" | 时效性，换场景可能失效 |
| **规则类** (rule) | 0.999/天 | ~2 年 | "不要删除 .git 目录" | 强约束，极少变化 |
| **关系类** (relation) | 0.99/天 | ~2 月 | "模块 A 依赖模块 B 的 UserService" | 随项目变化 |

**分类的用途：**
- 检索时可按类型过滤（"只查规则"或"查知识"）
- 衰减时不同类型不同速度（知识永久保留，经验快速淘汰）
- 压缩归档时保留优先级：规则 > 知识 > 习惯 > 关系 > 经验

## 四、子Agent 专业知识库

星空虫族中，不同兵种有不同的作战本能。工虫有筑巢知识，兵虫有战斗知识。对应到 Bugs——**不同类的子Agent 有自己专属的专业记忆**。

### 专业记忆的存储

```
完整场景记忆（50 MB）
    │
    ├── 通用记忆（所有Agent 可访问）
    │    ├── 知识类
    │    ├── 习惯类
    │    └── 关系类
    │
    ├── 浏览器子Agent 专业记忆（仅浏览器类Agent 检索）
    │    └── 反检测经验、搜索策略、常用网站结构
    │
    ├── 文件操作子Agent 专业记忆
    │    └── 路径约定、编码规则、权限模式
    │
    ├── 推理子Agent 专业记忆
    │    └── 算法设计模式、调试经验、性能优化笔记
    │
    └── ...（其他子Agent 类型）
```

### 实现方式：标签路由

```
每条记忆 → subagent_type: "browser"  # 可选标签
    ↓
浏览器子Agent 检索时 → 过滤 subagent_type = "browser" + "general"
文件子Agent 检索时 → 过滤 subagent_type = "file_op" + "general"
```

**不是物理隔离**，是检索时的标签过滤。同一场景中所有子Agent 共享同一场景记忆，但检索结果按专业过滤。

### 禁忌：专业记忆的信任值特点

| 专业记忆类型 | 初始信任 | 衰减 | 原因 |
|:----------|:------:|:---:|:----|
| 浏览器专业 | 0 | 0.95/天 | 反检测经验时效性强 |
| 文件操作专业 | 5 | 0.999/天 | 文件系统知识稳定 |
| 推理专业 | 0 | 0.99/天 | 经验可能过时 |
| 跨Agent 通用 | 0 | 见知识分类 | 各类型各自衰减 |

## 五、信任机制

| 维度 | 规则 |
|:----|:----|
| 初始值 | 0（不确定） |
| 增强 | 同一条记忆被不同来源交叉验证 |
| 减弱 | 被证伪、被更高优先级覆盖 |
| 上限 | +∞（绝对信任） |
| 下限 | -∞（绝对否定） |
| 去重 | 相同/相似内容合并，保留来源标签链 |
| 衰减 | 按知识分类不同速度衰减 |

## 六、记忆标签（完整版）

```
{
  agent_id: <首次报告的Agent ID>,
  scene_id: <记忆所属场景>,
  scope: "single" | "all" | ["s1", "s2"],  // 适用范围
  owners: ["MainAgent", "SubAgent#3"],    // 拥有此记忆的Agent
  category: "knowledge" | "habit" | "experience" | "rule" | "relation",
  subagent_type: "browser" | "file_op" | "reasoning" | "general",
  enhancement_rate: <+X/天>,              // 增强速率
  decay_rate: <-X/天>,                    // 衰减速率
  strength_cached: <当前强度>,            // 实时计算，仅作缓存
  last_updated: <timestamp>,
  last_validated: <timestamp>,
  validation_count: <次数>,
  source_count: <独立来源数量>,
  seq: <次序>
}
```

### scope 说明

| scope | 含义 | 示例 |
|:------|:----|:----|
| "single" | 仅此场景可见 | project-a 的 API 约定 |
| "all" | 所有场景可见 | 用户偏好 Tab 缩进 |
| ["s1","s2"] | 指定场景列表可见 | 跨项目共用的库约定 |

### owners 说明

| owners | 含义 |
|:-------|:----|
| ["MainAgent"] | 仅主Agent 可见 |
| ["MainAgent", "SubAgent#3"] | 主Agent + 特定子Agent |
| ["SubAgent:file_op"] | 所有文件操作类子Agent 可见 |
| ["AllAgents"] | 全部 Agent 可见 |

## 七、定期主动用户验证

定期询问用户以确认记忆准确性：

```
频率: 默认每天一次（可配置 h/m 级）
触发: 用户空闲时
选取: 当前场景 strength 最低的 N 条记忆
询问: "这条记忆还准确吗？" → 用户确认/否定/修正/跳过

详细设计见 16-信任引擎设计.md 第五章。

## 八、内存与性能预算

### 运行时内存（各层级独立）

| 组件 | 空闲 | 典型 | 峰值 |
|:----|:---:|:---:|:---:|
| 核心引擎（调度器+网关+Agent） | < 500KB | ~2MB | ~10MB |
| 向量索引（10 万条×768 维） | 0 | ~300MB | ~500MB |
| 加载的场景完整记忆 | 0 | ~50MB | 50MB |
| 浏览器内核进程 | 0 | ~200MB | ~300MB |
| **总计** | < 500KB | ~550MB | ~860MB |

> "极端低资源"指的是核心引擎。向量索引/记忆加载是按需的。

### 核心操作计算成本

| 操作 | 单次 | 10 万条批量 |
|:----|:---:|:---------:|
| 向量检索 | ~1ms | hnsw O(log N) |
| 相似度去重 | ~0.1ms | 内积 O(768) |
| 信任增减 | < 0.01ms | 纯算术 |
| 批次衰减扫描 | — | ~50ms（异步，不阻塞） |
```
