## 三、BUG.md 格式（Agent 规则文件）

```yaml
---
# YAML 头（可选）
name: "我的 Bugs Agent"
model: "claude-sonnet-4"
role: "coding"
---

# 你是一个专注于 Rust 系统编程的 AI 助手。

## 规则
- 所有代码使用 Rust，2018 edition
- 测试框架用 `cargo test`
- 不要写 Python 代码
```

| YAML 可配置字段 | 说明 |
|:--------------|:----|
| name | Agent 显示名称 |
| model | 指定使用的模型 |
| role | 工作模式：coding / research / general |
| tools.allow | 覆盖全局工具白名单 |
| tools.deny | 覆盖全局工具黑名单 |

## 四、热重载

```
修改任意配置文件 → filesystem watcher 检测（inotify/polling）
    ↓ 防抖 500ms
    ↓ 重载变更的配置
    ↓ 推送 config:changed 事件给各模块
    ↓ 模块自己决定是否应用新配置
        ├── scheduler → 即时生效（改参数）
        ├── tools → 即时生效（改权限）
        ├── memory → 即时生效（改阈值）
        └── network → 需要手动重启（端口绑定无法热切换）
        ├── models.base_url → 需要手动重启（连接池重建）
        ├── models.api_key → 即时生效
        └── agent model assignment → 即时生效
```

## 五、模型配置

### 三种配置方式

| 方式 | 流程 | 是否需要重启 |
|:----|:----|:----------:|
| **WebUI/GUI 配置** | 设置页面 → 表单填写 base_url + Token → 保存 | ❌ 即时生效 |
| **TUI 交互配置** | `bugs config set models.providers.xxx` → 交互式填写 | ❌ 即时生效 |
| **直接编辑 config.json** | 手动修改文件 → 保存 | ⚠️ base_url 变更需重启，api_key 即时生效 |

WebUI/GUI 和 TUI 是推荐方式，文件编辑是 fallback。

### 模型配置结构

```json
{
  "models": {
    "providers": {
      "anthropic": {
        "base_url": "https://api.anthropic.com",
        "api_key": "${ANTHROPIC_API_KEY}",
        "models": ["claude-sonnet-4", "claude-haiku-3.5"]
      },
      "openai": {
        "base_url": "https://api.openai.com/v1",
        "api_key": "${OPENAI_API_KEY}",
        "models": ["gpt-4o-mini"]
      },
      "ollama": {
        "base_url": "http://localhost:11434",
        "api_key": "",
        "models": ["qwen2.5-coder:7b", "llama3.2:3b"]
      }
    }
  },
  "agents": {
    "main": { "model": "claude-sonnet-4" },
    "subagent_defaults": {
      "models": {
        "reasoning": "claude-sonnet-4",
        "code": "qwen2.5-coder:7b",
        "fast": "gpt-4o-mini",
        "tiny": "llama3.2:3b"
      }
    }
  }
}
```

### 安全性

| 规则 | 说明 |
|:----|:----|
| api_key 用环境变量 `${VAR}` 引用 | 不把密钥明文写进 config.json |
| WebUI/GUI 输入 Token → 写入环境变量文件 | 不直接写入 config.json |
| Token 在日志中脱敏 | 输出 `sk-****` 而非完整密钥 |

## 六、假性能自检

| 错 | 配置有没有？ | 说明 |
|:---|:---------|:----|
| 固定权重 | ❌ | 权重可配置 |
| 两阶段截断 | ❌ | 无截断 |
| 分段跳跃 | ❌ | 配置都是连续值 |
| 立即重试 | ❌ | 无关 |
| 无上限 | ❌ | memory 有上下限，scheduler 有上限 |
