# Agent 角色

---

## 一、星空虫族的协作模式

星空虫族（Starship Troopers Arachnids）的战争哲学：

```
Overmind (Brain Bug) ← 战略思维、决策规划、指令下达
    │
    ├── 兵虫 (Warrior Bug) #1 → 执行攻击任务
    ├── 兵虫 (Warrior Bug) #2 → 执行侦察任务
    ├── 兵虫 (Warrior Bug) #3 → 执行防御任务
    └── 兵虫 (Warrior Bug) #N → 并行执行各类任务

特点：Overmind不需要亲自去撕咬敌人，它的价值在于——组织、协调、指派。
```

这种**"主控+并行执行"** 的模式就是 Bugs 的 Agent 架构。

## 二、Bugs 的 Agent 体系

| 中文名 | English | 类型 | 数量 | 说明 |
|:-----|:-------|:---:|:---:|:----|
| **主Agent** | Main Agent | AI Agent | 1 | 用户直接交互的入口，Overmind角色。负责理解意图、规划任务、生成子任务 |
| **子Agent** | SubAgent | AI Agent | 1+N 池 | 主Agent 派遣的小型 AI Agent，兵虫角色。执行具体的子任务，完成任务后汇报并回收 |
| **调度引擎** | Scheduler | 程序 | 1 | 子Agent 池管理器。创建/分配/回收/容错。类比虫后 |
| **记忆网格** | Memory Mesh | 程序 | 1 | 主Agent 和所有子Agent 共享的记忆空间 |
| **工具集** | Toolbox | 程序 | N | 主Agent 和子Agent 都可调用的能力库 |

## 三、主Agent：Overmind

| 维度 | 说明 |
|:----|:----|
| **职责** | 接收用户输入，理解意图，制定计划，分裂任务，派遣子Agent |
| **行为** | 由 BUG.md + ROLE.md 配置 |
| **不做什么** | 不执行具体的工具调用（那是子Agent 的活） |
| **资源** | 常驻，一个 LLM 实例 |

主Agent 是用户对话的入口。用户说"帮我查一下项目A的代码，修复所有lint错误，然后更新文档"——主Agent 把这个拆成 3 个子任务，各派一个子Agent 去干。

## 四、子Agent：兵虫

| 维度 | 说明 |
|:----|:----|
| **职责** | 执行一个具体子任务（如"检查项目A所有文件的lint错误"） |
| **生命周期** | 主Agent 派遣 → 执行 → 汇报结果 → 回收 |
| **资源** | 可为轻量 LLM 调用，也可不与 LLM 直接对话而仅调用工具 |
| **隔离** | 子Agent 之间不通信，各自独立执行 |
| **并行** | 多个子Agent 可同时执行（调度引擎管理并发） |

子Agent 是执行单元。它们不是常驻的——主Agent 说"派 3 个兵虫去干这三件事"，调度引擎创建 3 个子Agent，干完就回收。这就是星空虫族战争模式：**一个大脑，一群兵虫，闪电推进。**

## 五、为什么不是多层级多 Agent

| 方案 | 问题 |
|:----|:----|
| 主Agent + 编排器 + 协调器 + 执行器 | 四重 LLM 实例，资源浪费，层级过度 |
| **主Agent + 子Agent 池** | 主Agent 分裂任务 → 子Agent 执行 → 汇报。简洁、高效、可并行 |

Bugs 选择第二种。子Agent 的任务分解是主Agent 的 LLM 推理结果，不是另一个"编排器 Agent" 的 LLM 推理。

## 六、架构

```
用户输入
    │
    ▼
┌──────────────────┐
│   主Agent (Overmind)  │  ← 唯一的常驻 AI Agent
│   意图理解 / 计划   │
│   任务分裂 / 派遣   │
└──────┬───────────┘
       │  子任务列表
       ▼
┌──────────────────┐
│   调度引擎 (虫后)   │  ← 纯程序
│   创建 / 分配 /    │
│   回收 / 容错      │
└──────┬───────────┘
       │
   ┌───┼───────────┐
   ▼   ▼           ▼
┌──────┐ ┌──────┐ ┌──────┐
│子Agent│ │子Agent│ │子Agent│  ← 可并行的执行单元（兵虫）
│#1    │ │#2    │ │#N    │
└──┬───┘ └──┬───┘ └──┬───┘
   │        │         │
   └────────┼─────────┘
            ▼
   ┌──────────────┐
   │  工具集 / 记忆 │
   └──────────────┘
```

## 七、与竞品 SubAgent 的差异

| 维度 | OpenClaw subagent | Claude Code subagent | Bugs 子Agent |
|:----|:-----------------|:-------------------|:------------|
| 模型 | 可配置 | 继承主 Agent | 与主 Agent 共享模型 |
| 并发 | 限制 8 个 | 限制若干 | **无硬限制**（虫群式扩缩） |
| 生命周期 | 有 timeout | 有 timeout | **任务完成即回收** |
| 独立性 | 较强（独立 session） | 较强 | **高度协作**（共享记忆网格） |
| 类比 | — | — | **星空虫族兵虫** |
