# 平台与前端

## 平台支持

| 设备 | 架构 | 支持前端 | 优先级 |
|:----|:----|:--------|:-----:|
| Linux | x86_64 | TUI + Web + GUI | P0 |
| Linux | arm64 | TUI + Web + GUI | P1 |
| Windows | x86_64 | TUI + Web + GUI | P0 |
| Windows | arm64 | TUI + Web + GUI | P2 |
| Android | arm64 | GUI (APK) 仅 | P1 |

## 三种前端

| 前端 | 定位 | 技术方向 |
|:----|:----|:--------|
| **TUI 终端** | 最轻量，服务器/SSH 场景 | 内嵌终端界面，同一进程 |
| **Web UI** | 零安装，开浏览器就用 | 内嵌 HTTP 服务器，原生 HTML/JS |
| **GUI 桌面** | 独立窗口，完整交互 | egui，独立进程 |
| **Android APK** | 手机原生 | Kotlin + Compose |

## 模块化分离

```
核心引擎（纯逻辑，无 UI）
    ↑ REST+WS API
    ├── Web UI（内嵌 HTTP 服务器）
    ├── GUI 桌面（独立进程，走 API）
    ├── TUI 终端（内嵌，同一进程）
    └── Android APK（独立进程，走 API）
```

任何前端都不直接依赖核心的内部逻辑，全部通过 API 通信。
